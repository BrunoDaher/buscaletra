GIT WINDOWS

Conceito O git mapeia os arquivos da pasta do projeto e sabe quais arquivos sofreram alterações bem como as linhas que sofreram alterações. Com isso ele cria uma lista e no momento de atualizar o arquivo (no ambiente de desenvolvimento - em nuvem) ele altera só o que foi alterado. Sem necessidade de carregar todo arquivo novamente. Trabalha com conceito de "Branches" (varias versões do mesmo código) trabalhando em paralelo.
Passo a passo
  1. Criar conta em Git Hub
  2. Baixar e instalar Git SCM
  3. Escolher uma pasta de projeto ou criar uma
Operações com git
  1. git status - mostra status dos arquivos da contexto
  2. git branch - retorna bracnh atual
  3. git branch nome- Cria uma branch com o nome
  4. Deletar Branches
    Local git branch --d
    Remoto git branch -dr<remote>/<branch>
  5. git update - atualiza
  6. git remote -v - retorna repositório destino gravado
  7. git confg --listv - retorna configurações do git
  8. git revert - reverte
  9. git add - adiciona arquivos ao git
  10. git add untracked -
  11. git patch - atualiza arquivos
  12. git diff - mostra diferenças entre versões de um mesmo arquivo
  13. git quit - sai do git
  14. git rm - remove arquivo da lista e da pasta do projeto
  15. git commit - grava alterações incluídas
Exercício

*Na pasta do projeto executar os comandos

  1. botão direito do mouse-> menu de contexto/GitBash here
  2. Configurar Git
    $ git config -global user.name "Bruno Daher" $ git config -global user.email "brunus9@gmail.com"
  3. Iniciar Git
    git init
  4. Conferir Git
    git status
  5. Conferir Branches
    git branch
  6. Adicionar Destino (repositório)
    git remote add nomeDoRepositorio https://github.com/BrunoDaher/testetricks.github.io.git
  7. Adicionar todos arquivos ao índice
    git add *
  8. Commit
    git commit m "nome do commit e infos add"
  9. Enviar para destino
    git push nomeRepositorio branchAtual
  10. git diff mostra diferenças entre versões de um mesmo arquivo
  11. git quit sai do git
